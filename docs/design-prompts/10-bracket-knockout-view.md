# Prompt 10 — Bracket / knockout view

**Set:** B (new screens for shipped features)
**Status:** Live at `GET /bracket/{classId}` (`public/templates/bracket.html`). Shipped 2026-07-03 (Phase 2 knockout engine) — this is the flagship feature and the highest-value redesign target in Set B.

## Prompt (copy-paste this into Claude Design)

```
Torrorèndum: Catalan voting competition for Torrons Vicens, warm brown/gold/cream palette, burgundy accent. This is the flagship new screen: the Phase 2 knockout bracket. Design a tournament bracket view showing rounds — quarts de final, semifinals, gran final — for a category, with the surviving torrons advancing visually round to round toward a single winner. Needs to work at two very different sizes: a full desktop view showing the whole bracket tree, and a mobile view that reasonably shows "your current round" with a way to see the full tree. Use burgundy exclusively for the eventual winner's path so it reads as inevitable in hindsight. This is the single highest-stakes screen in the product — it should feel like the finals of something real, not a wireframe with lines connecting boxes.
```
